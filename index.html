Commit messages

Each time you push your changes back up to Github, you must include a commit message that details the changes you made.

commiting you work is writing a summary of your work

Rules for commiting messages

  • Minimum 3 words & 10 characters
  • Proper spelling & grammar
  • Capital starting letter, no period
  • Must start with an imperative verb: Create, Fix, Add, Solve, etc.
  • Pretend every commit starts with the phrase: “This commit will…”

Structuring content

Organizing our HTML elements content within typical regions of a web page.

header

  • The header is the introductory content usually at the top of the page and would include a logo or company name and main navigation. Headers are also within sections and articles to introduce the content for headings, subheadings, author, etc..

footer

  • Many webpages end their pages with a footer at the bottom. This often include lists of links, copyright information, extra information about the company,terms and services.

main

  • The main element wraps around the main content of the page. It’s only used once per page and it tells the browser that this is where the main content is.

section

  • Section is used to wrap around related pieces of content. Usually each section has its own heading.

article

  • The article element represents a self-contained composition in a web page, which is intended to be independently distributable or reusable. Examples include blog post, product card, interactive widget, etc...

nav

  • The nav element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents.

aside

  • The aside element represents a portion of a document whose content is only indirectly related to the document's main content. This content will frequently be presented as a sidebar on a website.

Landmark roles

ARIA has a set of landmark roles to help those using screen readers to jump directly to specific sections within our sites.

header role="banner"

  • This role is for defining an element as the masthead, added to the primary header.

nav role=“navigation”

  • This role is for defining an element as the primary navigation, added the the top nav.

main role=“main”

  • This role is for defining an element as being the main content of the page, added to the main.

aside role=“complimentary”

  • This role is for defining an element as being content related to the primary content, usually added to an aside.

footer role=“contentinfo”

  • This role is for defining an element as information about the content: e.g. copyright, terms, privacy policy, usually added to a footer.

Skip Links

Skip links are placed at the top of your page and offer users a way to jump to specific areas of the page.

Font Properties

Font Size

  • For adjusting the size of the font: a single number with a unit (rem [the best choice], em, px)

Font Weight

  • Sets how thick or thin characters in text should be displayed Ex: bold, normal, 700

Font Style

  • Specifies font style for text. Ex: italic or normal

Line Height

  • For adjusting the line-height: a single number, a multiplier that multiplies the font-size Ex: 1.5

Text Decorations

text-transform

  • Change capitalization such as uppercase, lowercase, capitalize. It’s a good idea to use CSS to make all caps instead of writing it directly in HTML for accessibility reasons. Ex: uppercase, lowercase, capitalize

text-decoration

  • Specifies the decoration added to text. The only things in a website that should be underlined are links. Ex: none, underline, line-through, overline

text-align

  • Specifies the horizontal alignment of text in an element Ex: left, center (American spelling), right, justify

text-indent

  • Indent the first line of text like traditionally printed materials Ex: 1rem, 2em, 20px, 25%

text-shadow

  • Adding a shadow to the text. Specified like this: text-shadow: horizontal-offset vertical-offset blur-radius colour text-shadow: 2px 2px #ff0000;

Padding

Padding adds spacing inside the box, pushing the content away from the border.

Margin

The margin is an invisible space around your box. It adds spacing outside of the box, pushing other boxes away from the box. Margins can have positive or negative values.
